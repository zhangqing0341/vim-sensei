1. Intro                                                        *sensei-intro*

vim-sensei 让你用自然语言描述一个编辑目标 (如 "删除到行尾"、"把双引号里的
内容整个替换掉"), AI 会返回地道的 vim 按键序列, 并把每个键拆开讲解, 给出
等效做法和举一反三的心法。对纯普通模式、可撤销的操作, 还能在光标处演示一次。

目标是让你"在实践中越来越会用 vim", 而不是替你按键。

==============================================================================
2. Requirements                                          *sensei-requirements*

- Vim 8+ (需要 |json_encode()| / |json_decode()|)
- `curl` 在 PATH 中
- 一个 OpenAI 兼容的 /chat/completions 接口 (OpenAI / LiteLLM / Ollama / ...)

==============================================================================
3. Setup                                                        *sensei-setup*

用你喜欢的插件管理器安装, 例如 vim-plug: >
    Plug 'zhangqing0341/vim-sensei'
<
最简单的方式: 装好后直接运行配置向导, 全程问答, 自动写好配置: >
    :SenseiSetup
<
向导三选一: 本地 Ollama (免 key) / OpenAI (填 key) / 自建网关。配置存到
|g:sensei_config_file| (默认 ~/.sensei.vim), 下次启动自动加载。

真·零配置: 若装了 Ollama 并拉了模型 (ollama pull qwen2.5-coder), 插件会
自动探测 localhost:11434, 什么都不用配、免 key、离线可用。

也可手动在 vimrc 配置。key 建议走环境变量 (默认 SENSEI_API_KEY): >
    export SENSEI_API_KEY="sk-..."
    let g:sensei_model = 'gpt-4o-mini'
<
自建 LiteLLM 网关示例: >
    let g:sensei_endpoint = 'http://127.0.0.1:4000/v1/chat/completions'
    let g:sensei_model    = 'claude-haiku-4-5'
<
本地 Ollama 示例: >
    let g:sensei_endpoint = 'http://127.0.0.1:11434/v1/chat/completions'
    let g:sensei_model    = 'qwen2.5-coder'
<

4. Usage                                                        *sensei-usage*

                                                                      *:Sensei* *:How*
:Sensei {自然语言}       描述一个编辑目标, 得到按键讲解。
:How {自然语言}          同上 (别名)。

                                                                 *:SenseiSetup*
:SenseiSetup             交互式配置向导 (选后端 / 填 key / 写配置文件)。

                                                                    *<Plug>(sensei-how)*
<Leader>ah               普通模式下弹出输入框, 输入目标。

示例: >
    :How 删除到行尾
    :How 复制当前这一整行
    :How 把双引号里的内容整个替换掉
    :How 选中当前段落
<

5. Configuration                                               *sensei-config*

                                                             *g:sensei_endpoint*
    OpenAI 兼容接口地址。默认 'https://api.openai.com/v1/chat/completions'。

                                                                *g:sensei_model*
    模型名。默认 'gpt-4o-mini'。

                                                         *g:sensei_api_key_env*
    读 API key 的环境变量名。默认 'SENSEI_API_KEY'。

                                                             *g:sensei_api_key*
    直接指定 key (优先于环境变量)。不建议写进版本管理的 vimrc。

                                                             *g:sensei_language*
    讲解语言。默认 '中文'。改成 'English' 即英文讲解。

                                                                 *g:sensei_demo*
    是否允许"演示一次"。默认 1。设 0 则只讲解不演示。

                                                          *g:sensei_map_default*
    是否创建默认映射 <Leader>ah。默认 1。设 0 则自行映射: >
        nmap <Leader>x <Plug>(sensei-how)
<
                                                          *g:sensei_config_file*
    :SenseiSetup 写入、启动时自动加载的配置文件。默认 '~/.sensei.vim'。
    该文件可能含明文 key, 请勿提交到版本库。

                                                            *g:sensei_ollama_url*
    本地 Ollama 地址, 用于自动探测。默认 'http://127.0.0.1:11434'。

                                                          *g:sensei_ollama_model*
    自动使用 Ollama 时的兜底模型名。默认 'qwen2.5-coder'。

                                                              *g:sensei_timeout*
    curl 超时秒数。默认 30。

                                                           *g:sensei_max_tokens*
    回复最大 token。默认 500。

==============================================================================
6. How it works                                             *sensei-internals*

纯 Vimscript, 无外部依赖 (除 curl)。请求体用 |json_encode()| 写入临时文件,
`curl -d @file` POST 到网关, |json_decode()| 解析。演示用 |feedkeys()|, 且仅
对 AI 标记为纯普通模式、可撤销 (safe_demo=true) 的按键开放, 避免误触发有副
作用的操作。

若用户未配置 key 且仍用 OpenAI 默认端点, 插件会自动探测本地 Ollama
(localhost:11434), 探到就免 key 直接用; 探不到则提示运行 :SenseiSetup。
:SenseiSetup 把配置写入 |g:sensei_config_file|, 该文件启动时被自动 source。
